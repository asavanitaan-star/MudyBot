# MudyBot — LuckyCondo LINE Bot

ผู้ช่วย LINE สำหรับ **agent ปล่อยเช่าคอนโด** — ดูแลผู้เช่าหลายห้องผ่านกลุ่มไลน์
โดยยึดหลัก **1 กลุ่มไลน์ = 1 ห้อง/สัญญา** มีระบบโปรไฟล์ห้อง คลังเอกสาร แจ้งซ่อม
ชำระค่าเช่า เตือนค่าเช่า/สัญญา และแดชบอร์ดรวมสำหรับ agent พร้อมตอบคำถามผู้เช่าด้วย AI

## ความสามารถ

- 🏠 **โปรไฟล์ห้องต่อกลุ่ม** — ผู้เช่า/เจ้าของ/ค่าเช่า/เงินประกัน/สัญญา/บัญชี กรอกทีเดียวทั้งชุด
- 📁 **คลังเอกสารต่อกลุ่ม** — เก็บรูปห้องก่อนเข้า/สัญญา/รูปตอนออก เรียกดูย้อนหลังได้
- 🛠️ **แจ้งซ่อม** — ผู้เช่าแจ้ง บอทออกเลขเคส แอดมินอัปเดตสถานะ (เด้งแจ้งกลับในกลุ่ม)
- 💰 **ชำระค่าเช่า** — แจ้งชำระ/ส่งสลิป → แอดมินยืนยัน (ยึดตามห้อง = กลุ่ม)
- 📊 **แดชบอร์ด agent** — สรุปข้ามทุกห้องในแชตเดี่ยว: ค้างชำระ / ซ่อมค้าง / สัญญาใกล้หมด
- 💸 **เตือนค่าเช่าอัตโนมัติ** — ตั้งค่าต่อกลุ่ม เตือนล่วงหน้าได้หลายรอบ (ตั้งด้วยภาษาธรรมชาติผ่าน AI ได้)
- ⏰ **เตือนสัญญาใกล้หมดอัตโนมัติ** — push หาแอดมินเมื่อเหลือ 60/30/7 วัน
- 🔘 **Rich Menu 2 ชุด** — เมนูผู้เช่า (ทุกคน) + เมนูแอดมิน (เฉพาะ ADMIN_IDS)
- 💬 **AI ผู้ช่วยดูแลผู้เช่า** — ตอบภาษาไทย (Gemini/Ollama) จำบทสนทนารายบุคคล ไม่ทำตัวเป็นเซลส์

## AI Provider ที่รองรับ

| Provider | วิธีใช้ | ข้อดี |
|----------|---------|-------|
| **Gemini** (แนะนำ) | `LLM_PROVIDER=gemini` | ฟรี ใช้งานบน cloud ไม่ต้องติดตั้งเพิ่ม |
| **Ollama** | `LLM_PROVIDER=ollama` | ฟรี ข้อมูลไม่ออกนอกเครื่อง แต่ต้องเปิดคอมไว้ |

## ความต้องการของระบบ

- [Node.js](https://nodejs.org/) 18+
- LINE Official Account + Messaging API channel
- ngrok (หรือ tunnel อื่น) เปิด webhook สู่อินเทอร์เน็ต — ใช้เสิร์ฟไฟล์ในคลังเอกสารด้วย
- Gemini API Key (ฟรี): [aistudio.google.com](https://aistudio.google.com) — หรือ Ollama ถ้ารันในเครื่อง

## ติดตั้ง

```bash
npm install
```

สร้างไฟล์ `.env`:

```env
LINE_CHANNEL_ACCESS_TOKEN=...
LINE_CHANNEL_SECRET=...

# AI provider: gemini หรือ ollama
LLM_PROVIDER=gemini
GEMINI_API_KEY=...
GEMINI_MODEL=gemini-flash-latest

# (ถ้าใช้ Ollama)
# OLLAMA_URL=http://localhost:11434
# OLLAMA_MODEL=qwen3:14b

# User ID ของแอดมิน (คั่นด้วย , ได้หลายคน) — พิมพ์ "ไอดีของฉัน" กับบอทเพื่อดู ID
ADMIN_IDS=Uxxxxxxxx,Uyyyyyyyy

# URL สาธารณะของ tunnel (ใช้เสิร์ฟไฟล์คลังเอกสาร) — ค่าเริ่มต้นชี้ไปโดเมน ngrok ที่ล็อกไว้
# PUBLIC_URL=https://your-static-url.ngrok-free.dev

PORT=3000
```

## รัน

```bash
npm start
# แล้วเปิด tunnel: ngrok http --url=<your-static-url> 3000
# ตั้ง Webhook URL ใน LINE: https://<your-url>/webhook
```

บน Windows ดับเบิลคลิก `start.bat` เพื่อเปิดทั้งบอทและ ngrok พร้อมกัน

## ระบบแอดมิน

- พิมพ์ `ไอดีของฉัน` กับบอท → ได้ LINE User ID นำไปใส่ `ADMIN_IDS` ใน `.env` แล้วรีสตาร์ต
- แอดมินใช้คำสั่งจัดการได้ทั้งหมด และได้ Rich Menu แอดมินอัตโนมัติ

## คำสั่งหลัก

> ในกลุ่มต้องขึ้นต้นด้วย `ลัคกี้` — ในแชตเดี่ยวพิมพ์ได้เลย

**ข้อมูลห้อง (ในกลุ่มของห้อง, แอดมิน)**

| คำสั่ง | ผล |
|--------|-----|
| `ตั้งข้อมูลห้อง` | รับฟอร์มเปล่า กรอกแล้วส่งกลับทั้งก้อน |
| `ดูข้อมูลห้อง` | ดูโปรไฟล์ห้อง (ทุกคนในกลุ่มดูได้) |

**เอกสาร (ในกลุ่มของห้อง)**

| คำสั่ง | ผล |
|--------|-----|
| `บันทึกสัญญา` / `บันทึกรูปก่อนเข้า` / `บันทึกรูปตอนออก` | เข้าโหมดเก็บไฟล์ แล้วส่งรูป/ไฟล์ |
| `ขอสัญญา` / `ขอรูปก่อนเข้า` / `ขอรูปตอนออก` | บอทส่งไฟล์กลับ |

**ผู้เช่า (ในกลุ่มของห้อง)**

| คำสั่ง | ผล |
|--------|-----|
| `แจ้งซ่อม <รายละเอียด>` | เปิดเคสแจ้งซ่อม |
| `แจ้งชำระแล้ว` / ส่งสลิป | แจ้งชำระค่าเช่าเดือนนี้ |
| `ดูสถานะชำระ` / `ดูการแจ้งซ่อม` | ดูสถานะของห้องตัวเอง |

**แดชบอร์ด & จัดการ (แอดมิน — ส่วนใหญ่ใช้ในแชตเดี่ยว)**

| คำสั่ง | ผล |
|--------|-----|
| `คำสั่งแอดมิน` | ดูคำสั่งทั้งหมด |
| `สรุปเดือนนี้` | สรุปการชำระทุกห้อง + ยอดค้างรวม |
| `ห้องทั้งหมด` | ลิสต์ห้องที่ดูแลทั้งหมด |
| `ค้างชำระ` / `ซ่อมค้าง` / `สัญญาใกล้หมด` | รายการที่ต้องตาม |
| `ยืนยันชำระ` | ยืนยันการชำระ (พิมพ์ในกลุ่มของห้อง) |
| `อัปเดตซ่อม #3 เสร็จแล้ว` | อัปเดตสถานะงานซ่อม |
| `ประกาศ <ข้อความ>` | ส่งประกาศเข้าทุกกลุ่ม |

**เตือนค่าเช่า (ในกลุ่ม)** — `ตั้งวันครบกำหนด 5`, `ตั้งเตือนล่วงหน้า 3,1 วัน`, `ตั้งเวลาเตือน 09:00`, `เปิดเตือน`/`ปิดเตือน`, `เตือนเดี๋ยวนี้`, `คำสั่ง`
หรือพูดภาษาธรรมชาติ เช่น "เตือนค่าเช่าทุกวันที่ 5 ตอน 9 โมง ล่วงหน้า 3 วัน"

## Rich Menu

เมนูปุ่มก้นจอ (แสดงเฉพาะแชตเดี่ยว) มี 2 ชุด: เมนูผู้เช่า (default) และเมนูแอดมิน (ผูกกับ `ADMIN_IDS`)

```bash
# 1) สร้างรูปเมนู (Windows PowerShell)
powershell -File tools/make-richmenu-image.ps1
# 2) ลงทะเบียน + ผูกเมนูแอดมิน
node tools/setup-richmenu.js
```

แก้ปุ่ม/สี/ข้อความได้ใน `tools/make-richmenu-image.ps1` และ `tools/setup-richmenu.js`

## การปรับแต่ง

แก้บุคลิก AI ช่องทางติดต่อ และข้อความต่าง ๆ ได้ในไฟล์ [`info.js`](info.js)

## โครงสร้างไฟล์

```
├── bot.js          # โค้ดหลัก (webhook, AI, คำสั่ง, reminder, เอกสาร, แดชบอร์ด)
├── info.js         # บุคลิก AI ช่องทางติดต่อ และข้อความต่างๆ
├── tools/
│   ├── make-richmenu-image.ps1   # สร้างรูป Rich Menu (ผู้เช่า + แอดมิน)
│   └── setup-richmenu.js         # ลงทะเบียน Rich Menu ผ่าน LINE API
├── start.bat       # เปิดบอท + ngrok พร้อมกัน (Windows)
└── package.json
```

> ไฟล์ข้อมูลขณะรัน (`units.json`, `payments.json`, `repairs.json`, `docs.json`, `media/` ฯลฯ)
> และ `.env` ถูก gitignore ไว้ — มีข้อมูลส่วนตัวผู้เช่า ไม่ควร commit ขึ้น git
```
